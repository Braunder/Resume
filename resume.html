Продуктовая компания (разработка LLM-решений для автоматизации текстовых запросов)
Дообучение доменных LLM
  • Адаптировал Qwen, Gemma, Llama (3K–30K записей) под предметную область через LoRA/QLoRA. Подбирал баланс между качеством и стоимостью обучения.
Результат
Точность релевантных ответов: 0.62 → 0.81 (внутренний бенчмарк, 500 примеров). Галлюцинации на доменных запросах снизились на ~40%.
RAG на 10K+ документах
  • Построил пайплайн на ChromaDB: гибридный retrieval (dense + sparse), реранкинг через cross-encoder (bge-reranker), фильтрация по метаданным.
  • Решал ограничения контекстного окна: чанкинг с оверлапом, отбор топ-K релевантных чанков по score реранкера, приоритизация наиболее информативных фрагментов в промпт.
  • Интегрировал в прод через LangChain / LlamaIndex.
Результат
Ответы стали привязаны к источникам. Снизилась потребность в дообучении модели под каждый новый документооборот.
Self-hosted инференс
  • Перевёл критичные сценарии с внешних API на собственные GPU: A100, H100, RTX 3060.
  • Оптимизировал через vLLM, llama.cpp: квантизация (GPTQ/AWQ), управление KV-cache, батчинг.
Результат
Latency: 2.5с → 0.8с на 70B-модели. Throughput вырос в 3.5×. Инфраструктурные затраты сократились примерно в 4×.
Агентные сценарии (до 3 шагов)
  • Разработал цепочки с function calling: поиск сайта → выжимка контента → запись в файл.
  • Реализовал self-correction: при ошибке инструмента агент перепланирует шаги, а не падает.
Результат
Автоматизировал комплексные запросы, которые раньше требовали ручной сборки из 2–3 разных сервисов.
Оценка качества и борьба с галлюцинациями
  • Собрал бенчмарк из реальных пользовательских запросов (не синтетика). Проводил A/B-тесты, human-in-the-loop.
  • Внедрил grounding через retrieval и верификацию ответов по цитатам из источников.
Результат
Hallucination rate в продакшене: 18% → 6%.
